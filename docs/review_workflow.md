# Review workflow

## Цель

Review-процесс в `docustruct-ai` предназначен для selective human review: человек проверяет только спорные поля, а не весь документ целиком.

## Когда поле попадает в review

Поле отправляется в review, если:
- confidence ниже `accept_threshold`, но выше `reject_threshold`;
- отсутствует grounding;
- validation вернула ошибку или warning;
- документ низкого качества;
- есть расхождение между несколькими engines.

## Что видит reviewer

- исходную страницу;
- значение-кандидат;
- статус валидации;
- evidence text;
- `bbox`, если grounding найден;
- сырой OCR text;
- альтернативные кандидаты, если они были рассчитаны.

## Возможные действия

- `accept`
  Подтвердить извлечённое значение без правок.

- `edit`
  Исправить значение вручную и сохранить его как финальное.

- `unsupported`
  Явно отметить поле или документ как неподдерживаемый.

## Что сохраняется

В систему пишутся:
- review task;
- финальное решение;
- reviewer;
- timestamp;
- комментарий;
- финальное значение.

Эта структура пригодна для последующего:
- анализа качества;
- построения reviewer analytics;
- генерации datasets для active learning.
